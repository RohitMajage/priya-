01 — Problem & Solution

What this platform solves

Criminal investigations today drown in unstructured multimedia evidence — CCTV footage, audio, photographs, digital documents — all managed through fragmented manual processes that break under scale and pressure.

⚠️

Forensic workflows are slow, fragile and siloed

Law enforcement agencies lack a unified intelligent platform to handle the growing volume of digital evidence. Manual processes create bottlenecks, errors and security gaps at every step of an investigation.

  • Manual cataloging of photographs, audio and documents is error-prone and time-consuming
  • Physical evidence vulnerable to tampering, misplacement and chain-of-custody failures
  • Existing tools are siloed — separate systems for face recognition, voice and storage
  • No biometric cross-referencing or pattern detection across cases
  • No centralized audit trail for legal accountability and admissibility
🚀

One intelligent, secure, unified forensic platform

A full-stack web platform integrating face and voice biometrics with secure evidence handling, automated classification, RBAC and real-time investigator dashboards — all in one Django application.

  • 128-D face embeddings via dlib CNN for real-time identity matching at 96% accuracy
  • 256-D voice embeddings via Resemblyzer (PyTorch) with MFCC preprocessing at 92% accuracy
  • RBAC for Admin, Investigator and Officer with full audit logging on every action
  • AES-256 encryption, HTTPS, CSRF, OTP multi-factor authentication — zero vulnerabilities
  • Real-time dashboard, exportable PDF reports and SMTP biometric match alerts

06 — Challenges & Learnings

Hard problems, real solutions

Every major technical obstacle encountered during development — and the engineering decisions that resolved them, turning each challenge into a durable system improvement.

CHALLENGE 01

Responsive UI across three user roles

Designing a frontend that worked for Admin, Investigator and Officer — each with different permissions and module access — across desktops, tablets and mobile devices simultaneously.

→ Role-based dynamic navigation using Django template conditionals + Bootstrap 5.3 responsive grid. Each role sees only their authorised modules.
CHALLENGE 02

Evidence upload validation & routing

The upload portal needed to accept .jpg, .mp3, .mp4, .pdf and route each to the correct biometric module while rejecting invalid formats — without exposing backend errors.

→ MIME type checking + Django FileField validation + custom routing logic. User-friendly error messages for all validation failures — no raw exceptions.
CHALLENGE 03

Dashboard performance under load

The dashboard slowed significantly when displaying large evidence lists and match results simultaneously — especially with filters on unoptimised queries.

→ Database indexing + lazy query evaluation via Django ORM + paginated evidence list views. Dashboard load time reduced substantially.
CHALLENGE 04

OTP email delivery reliability

During testing, OTP emails were occasionally delayed or marked as spam — causing failed logins during time-sensitive investigative scenarios.

→ Configured SMTP with proper email headers and sender authentication. Added clear UI feedback during OTP wait ("valid for 30 minutes"), reducing confusion.
CHALLENGE 05

Cross-browser form accessibility

Form elements (file uploads, dropdowns, multi-select viewer permissions) rendered inconsistently across Chrome, Firefox and Edge — especially on the evidence upload page.

→ Standardised form components using Bootstrap 5.3 with explicit ARIA labels. Tested across all major browsers — confirmed consistent keyboard accessibility.
CHALLENGE 06

Flask-to-Django ML integration

Biometric modules developed in Flask created friction when brought into Django — SQLAlchemy conflicts, import path issues and response format mismatches.

→ Refactored biometric modules as standalone Python callable services, fully decoupled from any ORM or web framework assumptions.
